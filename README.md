## Frameworks Recomendados:
JavaFX: Para criar a interface gráfica de forma moderna e intuitiva.
JavaFX CSS: Suporte nativo ao CSS, permitindo que você estilize seus componentes de interface gráfica utilizando arquivos *.css
Hibernate: Para facilitar a manipulação do banco de dados, se você decidir usar um.
JDBC (Java Database Connectivity): Para conexão direta com o banco de dados, caso não utilize Hibernate.
JUnit: Para testes automatizados das funcionalidades do seu código.
Log4j: Para registro de logs e facilitar a depuração do sistema.


# Sprint 1: Preparação e Estruturação do Projeto
Duração: 2 semanas

## Tarefas:

### Configurar o Projeto em IntelliJ e Criar Repositório Git:
Descrição: Inicie um novo projeto Java no IntelliJ e configure um repositório Git.
Duração: 1 dia.

### Desenvolver Protótipo no Figma:
Descrição: Crie um protótipo da GUI com as telas principais (Login, Tela Principal, Cadastro de Doações).
Duração: 4 dias.

### Definir Estrutura de Classes:
Descrição: Esboce as classes principais do projeto (ex: Doador, Doacao, SistemaDeDoacoes).
Duração: 2 dias.
Implementar Classes de Modelo:

### Descrição: Codifique as classes definidas anteriormente.
Duração: 3 dias.
Configurar Banco de Dados (opcional):

### Descrição: Escolha uma solução de banco de dados (ex: SQLite) e configure a conexão.
Duração: 2 dias.
Revisão e Documentação Inicial:

### Descrição: Faça uma revisão das implementações e escreva uma documentação inicial do projeto.
Duração: 1 dia.


# Sprint 2: Desenvolvimento das Funcionalidades Principais
Duração: 2 semanas

## Tarefas:

### Implementar Tela de Login:
Descrição: Crie a interface e a lógica de autenticação.
Duração: 2 dias.
Desenvolver Tela Principal:

### Descrição: Construa a tela que exibe as doações e a opção para adicionar novas.
Duração: 3 dias.

### Implementar Cadastro de Doações:
Descrição: Crie a interface e a lógica para adicionar doações ao sistema.
Duração: 3 dias.

### Implementar Rastreamento de Contribuições:
Descrição: Crie uma funcionalidade para listar doações com opções de edição e exclusão.
Duração: 4 dias.

### Enviar Agradecimentos:
Descrição: Desenvolva a interface e lógica para enviar mensagens de agradecimento.
Duração: 3 dias.

### Testes de Usabilidade:
Descrição: Realize testes com usuários para feedback sobre a interface.
Duração: 2 dias.

### Revisão e Refatoração:
Descrição: Revise o código e faça melhorias baseadas no feedback recebido.
Duração: 1 dia.
